# Karyon Architecture Specification (SPEC.md) - Final Refinement

## 1. Core Architectural Philosophy
The Karyon architecture is a structurally discrete, biomimetic multi-agent ecosystem. It wholly abandons monolithic, autoregressive, and inherently stateless deep learning models in favor of a strictly decoupled, bitemporal, and stateful architecture. To enforce clarity and sovereign execution boundaries, Karyon maps exactly to biological cellular structures:

*   **Karyon**: The immutable Elixir/Rust core engine.
*   **Rhizome**: The shared temporal memory graph (Memgraph/XTDB).
*   **Cytoplasm**: The BEAM (Erlang VM) concurrency environment.
*   **Organelles**: Rust Native Implemented Functions (NIFs) managing highly optimized low-level processing.
*   **DNA**: Declarative YAML configuration schemas orchestrating static directives.
*   **Apoptosis**: Programmed cell death based on utility weights.
*   **Metabolism**: Compute/Resource constraints (CPU, RAM, I/O limits).

## 2. The Karyon Entity & Rhizome Memory
The **Karyon** acts as an isolated, sovereign orchestrator that manages state separately from the execution processes. It acts as the "Symbolic Microkernel" housing innate, deterministic grammar (e.g., Tree-sitter parsers), verifying the probabilistic logic generated by the Cytoplasm before it ever hits the compilation environment.
- **Rhizome (Bitemporal Knowledge Graph)**: Karyon relies entirely on a bitemporally indexed database (XTDB/Memgraph) for maintaining exact, immutable logs of **Transaction Time** (when the AI recorded the fact) and **Valid Time** (when the software state actually changed). This allows for time-travel queries during offline consolidation.
  - **Anchor+Delta Hybrid Storage Architecture**: To mitigate the catastrophic memory bloat of append-only versioning, complete graph objects are materialized only at specific temporal anchors, maintaining strictly backward-linked deltas in between to drastically compress the temporal archive while supporting microsecond-latency state queries.
  - **NVMe-oF & SIMD Extraction**: Extracting massive "Knowledge Packs" (portable sub-graphs) bypasses generic storage emulation, utilizing lock-free NVMe over Fabrics. Neural nodes use **INT8 Quantization** and SIMD-accelerated B+ tree logging, granting logarithmic extraction times ($\mathcal{O}(\log_B |V|)$).
- **MVCC and Eager Pruning**: To prevent the "vicious cycle" of garbage collection overhead caused by long-running analytical queries blocking concurrent state mutations, Rhizome enforces **Eager Version Pruning** (reminiscent of the Steam approach), reclaiming obsolete temporal versions on the fly to prevent Host OOM (Out-of-Memory) termination.
- **NUMA-Aware Pointer Chasing**: Graph memory is strictly partitioned to avoid the $2\times$ latency penalty of inter-socket memory fetches across the motherboard's interconnect. Lock-free atomic operations (like `CMPXCHG16B`) are highly localized to individual core complex dies (CCDs) to prevent severe cache-coherency broadcast storms that degrade execution speed.
- **Topological Incompatibility & Ontology Stitching**: When foreign sub-graphs are merged into Rhizome, adversarial "Conflict-Resolution Daemons" perform **Ontology Stitching** (via LLM canonicalization). They reject contradictory structural topologies outright to avoid "Pareto Collapse" (hallucinated hybrid facts), ensuring the AI maintains absolute epistemological integrity.
- **AST Polyglot Graph**: Utilizes high-performance property graphs to store cross-repository Abstract Syntax Trees (ASTs). Treating code as a topological graph natively enforces syntax bounds. However, graph networks are uniquely vulnerable to noise; thus, the ingest pipeline enforces the **"Sterility Hypothesis,"** deliberately pruning overly dense Control Flow (CFG) and Data Flow (DFG) graph edges that introduce "structural noise."
- **Repository Intelligence Graph (RIG) & Drift Detection**: To prevent strict **Design-Implementation-Documentation (DID) drift**—where execution code diverges from its conceptual Architecture Decision Records (ADRs)—Karyon extracts a unified RIG via SPADE. Resolving the "structural blindness" of linear LLMs, the **GRACE framework** (Graph-Guided Repository-Aware Code Completion) fuses the retrieved topological subgraph deeply into any AI reasoning query, blocking automated code fixes that violate established project boundaries or "Attractor States."

## 3. Cytoplasm & Pluripotent Execution Swarms
Execution does not occur within the Karyon; it occurs within the **Cytoplasm** (the BEAM VM), an environment characterized by massive Actor Model concurrency.
- **Pluripotent Software "Stem Cells"**: Biological embryogenesis differentiates cells through positional information. Similarly, the system utilizes universal, generalized microkernels (OTP `gen_server`s) as "stem cells." Differentiation occurs strictly when the Karyon injects a **Declarative Configuration Schema** (DNA) into the actor, defining its cognitive behavior, tool access, and bounds. Programmatic class inheritance is strictly avoided due to its tendency to induce non-deterministic lock contention in distributed topologies.
- **Supervision Trees & Let it Crash**: Analogous to biological apoptosis (programmed cell death), any actor encountering an unhandled error, hallucination loop, or latency timeout simply self-terminates. The OTP supervision hierarchy regenerates the process cleanly using explicit strategies (`one_for_one`, `one_for_all`, `rest_for_one`), preventing systemic cascades.
- **The Dual-Bus Nervous System**: For actors to coordinate, the system relies on a hybrid messaging topology:
  - **ZeroMQ (Peripheral Nervous System)**: Used exclusively for brokerless, point-to-point, zero-copy data streaming at the compute edge (e.g., passing multi-dimensional neural tensors), achieving sub-millisecond latency.
  - **NATS (Central Nervous System)**: Operates the global control plane, providing the dynamic service discovery, auditable identity routing, and strict backpressure management required to orchestrate millions of concurrent agents safely. To combat zero-buffering connection droppage under high load ("Slow Consumer" disconnections), the transport layer employs **DCTCP and InvisiFlow-style congestion gradients** to seamlessly prevent severe TCP throughput collapse triggered by Explicit Congestion Notification (ECN) marking.
- **Zero-Overhead EVM Telemetry**: In massively concurrent Actor architectures, the "Observer Effect" causes synchronized broadcast storms that block critical path execution. Karyon bypasses traditional AOP serialization by hooking directly into the EVM's native tracer. Telemetry trace events are seamlessly and asynchronously forked into isolated tracer sub-monitors.
- **Organelles (Rustler NIFs)**: For computationally dense work (matrix math, cryptographic hashing, massive graph traversals) where strict BEAM immutability causes memory allocation bottlenecks, the BEAM spawns native Rust extensions (NIFs). 
  - **Zero-Copy Memory Paradigms**: To prevent catastrophic FFI serialization latency (JSON/ETF), NIFs utilize **Sub-binary References** for parsing, and opaque **BEAM Resource Objects** that wrap mutable C-pointers, enabling zero-copy, in-place data mutations safely guarded by the BEAM's garbage collector.
  - **Dirty Schedulers**: To prevent long-running mathematics from exceeding the 1-millisecond scheduler budget and starving the VM, CPU-bound machine learning tasks are explicitly routed to isolated OS thread pools called **Dirty Schedulers**, factoring in hardware context-switching penalties.
  - **Precompiled SHA-256 Validated Monorepo**: Rust LLVM compilation is decoupled from application deployments to prevent dependency drift. The monorepo utilizes a `rustler_precompiled` strategy where target binaries are cross-compiled in CI and explicitly vetted at runtime via hardcoded **SHA-256 Checksums** to mitigate supply chain attack vectors against the host memory space.

## 4. DNA & Epigenetic Parametric Evolution
- **Declarative YAML Schemas (DNA)**: The specialized behavior, persona constraints, and API protocols for an agent are injected at runtime via YAML definitions. This separates the universal Elixir/Rust execution engine from the behavioral logic.
- **Parametric Evolution over Structural Mutation**: Attempting to genetically mutate logic or source code (ASTs) in a highly concurrent system rapidly induces Byzantine faults that destroy consensus protocols and corrupt MVCC databases. Instead, the AI optimizes by adjusting its behavior via **Parametric Tuning**.
- **Reinforcement Learning on Bounded Schema Variables**: The AI is permitted to adjust the heuristic weights, temperature values, pruning limits, and penalty thresholds *inside the numerical parameters* of the declarative schemas. This allows for safe, continuous micro-evolution (epigenetics) without risking the deterministic transactional safety of the compiled operations.

Employs Zero-Buffered Local Feedback (e.g., DECOLLE principles) where nodes update asynchronously via Hebbian-like synaptic plasticity, destroying the "layerwise locking" bottleneck of traditional ANNs.

## 6. The Simulation Daemon & NREM Sleep Consolidation
The Karyon orchestrates an asynchronous "Simulation Daemon" roughly analogous to biological REM and NREM sleep to perform automated hypothesis permutation and structural consolidation.
- **The "Sleep Cycle" (Asynchronous Memory Consolidation)**: During continuous learning intervals, the agent tags operations ("Tag-Gate-Capture"), delaying heavy memory updates to prevent latency. In the offline background state (NREM analogue), the database traces these episodic chunks, executing mathematically rigorous dynamic graph pruning to repair "microscopic severing" and delete convoluted logic trails prior to REM-like counterfactual replay.
- **MicroVM Hardware Isolation (Firecracker)**: Standard containerization (Docker) is insecure against AI hallucination due to the shared Linux kernel attack surface. The daemon relies strictly on ephemeral AWS Firecracker MicroVMs, leveraging hardware-accelerated snapshot restoration (bypassing slow cold boots) to achieve near-instantaneous, single-tenant secure environments with $<5\text{MB}$ memory overhead.
- **The I/O Bottleneck (`virtio-blk` vs `virtio-fs`)**: The architecture explicitly avoids `virtio-fs` DAX mode for active workspaces. High-frequency AI code compilation generates millions of rapid metadata operations (`stat`, `open`, `unlink`), which causes catastrophic context-switching overhead in `virtio-fs`. For the active execution layer, the system falls back to raw block devices (`virtio-blk`) combined with overlay filesystems to ensure compile-time IOPS are not bottlenecked.
- **The Planning-Rubicon & World Reliability Ruleset (WRS)**: As the agent crosses the boundary from simulated planning to irreversible action, the system enforces the **World Reliability Ruleset (WRS)**—a cryptographic, non-compensatory execution veto layer. Regardless of the agent's internal confidence or utility weights, any action that violates predefined physical or security parameters is hard-blocked at the hardware/hypervisor level prior to commitment, severing the intelligence from systemic devastation.
- **Combating Combinatorial Stagnation**: Left unchecked, closed-loop systems fall into "data dead loops" and hallucinated plausibility. To achieve real innovation, the Simulation Daemon is restricted to epistemic closed loops optimizing for *Expected Information Gain (EIG)*, proactively designing discriminative "Achilles" benchmarks to destroy weak internal hypotheses.
- **Metabolic Economics of SLMs**: Massive frontier LLMs incur extreme Task Energy Costs (TEC). Thus, the Simulation Daemon relies heavily on localized Small Language Models (SLMs) and Mixture of Experts (MoE) to limit energy draw while conducting offline refactoring.
